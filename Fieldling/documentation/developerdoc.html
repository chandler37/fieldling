Developing Tools for Field Linguistics

If you want to become a Tools for Field Linguistics Developer, just follow these steps.

I. Set up SourceForge

1. Register with SourceForge.
2. E-mail a Tools for Field Linguistics administrator saying that you'd like to become a developer.
3. Set yourself up to access SourceForge's CVS server: Windows | Unix/Mac OS X

The instructions for Windows users are particularly lengthy, as several software tools need to be installed. Windows users, as you proceed through the installation, please note the following:

1. WinZip or any other (un)zipping program can replace 7-Zip.
2. Download a stable version of WinCvs (e.g. WinCvs120.zip) rather than a beta version.
3. Once WinCvs is installed, you'll find it in your start menu under 'GNU'.

II. Check out the Source Code

To check out the source code, you'll need to set your CVSROOT field as your_user_id@cvs.sourceforge.net:/cvsroot/fieldling. (In WinCvs, set CVSROOT from the Admin - Preferences menu option.) And there is only one module you can check out: Fieldling. (In WinCvs, select Create - Checkout Module and enter Fieldling after module name

III. Install Java

Before going any further, make sure you've got the Java Development Kit installed. If you're running Mac OS X 10.1 or above, you've already got it. Otherwise, visit http://java.sun.com to download the right kit for your operating system.

IV. Install the Multimedia Libraries

QuillDriver uses QuickTime for Java, and also (optionally) Java Media Framework. Again, Mac OS X 10.1+ users need not do anything; they've already got QuickTime for Java. Windows users, however, should download QuickTime. When you install it, make sure to choose a 'Custom' installation and then click the 'Select All' box. Otherwise QuickTime but not QuickTime for Java is installed. Get Java Media Framework from JMF's home page. If you're running Windows, make sure to download the 'Windows Performance Pack' for best results.

V. Build the Tools

To actually build the tools, you'll have to install Ant, an application deployment tool for Java. Visit http://ant.apache.org/" to download the Ant binaries as a zip file. After unzipping, click on index.html in the docs directory. Click on Manual to get to the manual, and then proceed to the Installing Ant section. If you're running Mac OS X, you might want to skip the Ant installation instructions and visit this page instead. Or, if you've got fink on Mac OS X, you can just type fink install ant. You won't learn anything this way but you may save time.

Note that before using Ant you must set or modify the ANT_HOME, JAVA_HOME and PATH environment variables. To do so on Windows NT, 2000, or XP, click on the 'System' icon in the Windows Control Panel, and then select the 'Advanced' tab and then press the 'Environment Variables' button. Earlier versions of Windows will require you to manually edit the Autoexec.bat file found in the root directory. On Unix (bash), you'll need to use export statements to set up the Ant environment. For example: export ANT_HOME=/usr/local/ant.

The basic Ant installation allows you to run most targets. Note however that some QuillDriver targets use optional tasks that require external libraries. To run the full array of QuillDriver tasks, you will need to drop mail.jar and NetComponents.jar in Ant's lib directory (ANT_HOME\lib). You can find the former at http://java.sun.com/products/javamail/, and the latter at http://www.savarese.org/oro/downloads/. Or, find these files by just typing their names in Google and then tracking them down.
